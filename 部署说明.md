# BC 公司申报管理系统 - NAS Docker 部署指南

## 项目信息

- **技术栈**: Next.js 15 + SQLite + Drizzle ORM
- **数据库**: SQLite，保存在 Docker volume 中
- **镜像**: `ghcr.io/zzfca/company-return-alert:latest`
- **访问方式**: 使用单用户账号登录

## NAS 部署步骤

现在不需要上传源码，也不需要在 NAS 上构建。GitHub Actions 会自动构建 Docker 镜像，NAS 只需要拉取镜像运行。

### 方式 A：Docker Compose（推荐）

1. SSH 登录 NAS：

```bash
ssh your-nas-username@nas-ip-address
```

2. 创建部署目录：

```bash
mkdir -p /volume1/docker/bc-company-manager
cd /volume1/docker/bc-company-manager
```

3. 创建 `docker-compose.yml`：

```bash
nano docker-compose.yml
```

4. 粘贴以下内容：

```yaml
services:
  app:
    image: ghcr.io/zzfca/company-return-alert:latest
    container_name: bc-company-manager
    restart: unless-stopped
    ports:
      - "3588:3000"
    volumes:
      - company-data:/app/data
    environment:
      NODE_ENV: production
      PORT: 3000
      DATABASE_URL: file:/app/data/db.sqlite
    healthcheck:
      test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:3000"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s

volumes:
  company-data:
```

5. 拉取镜像并启动：

```bash
docker compose pull
docker compose up -d
```

6. 查看日志：

```bash
docker compose logs -f
```

7. 浏览器访问：

```text
http://你的NAS的IP:3588
```

默认账号：

| Username | Password |
| --- | --- |
| admin | admin |

登录后可在右上角修改密码。

## 数据持久化

数据库路径为容器内 `/app/data/db.sqlite`，通过 Docker volume `company-data` 保存。

- 更新镜像不会删除数据。
- `docker compose down` 不会删除数据。
- `docker compose down -v` 会删除数据库 volume，请谨慎使用。

## 更新应用

当 GitHub Actions 发布新镜像后，在 NAS 部署目录运行：

```bash
docker compose pull
docker compose up -d
```

## 端口配置

默认外部访问端口为 `3588`。如需改为 `8080`：

```yaml
ports:
  - "8080:3000"
```

## 反向代理

如果通过域名访问，把反向代理目的地指向：

```text
localhost:3588
```

如果使用群晖 DSM，可在 **控制面板 → 登录门户 → 高级 → 反向代理服务器** 中配置。

## 常用命令

```bash
# 查看容器
docker ps

# 查看日志
docker compose logs -f

# 停止
docker compose down

# 重启
docker compose restart

# 更新
docker compose pull
docker compose up -d
```

## 故障排查

### 镜像无法拉取

```bash
docker pull ghcr.io/zzfca/company-return-alert:latest
```

如果提示权限错误，请到 GitHub 仓库的 Packages 页面把 `company-return-alert` package 设置为 public。

### 无法访问或无法写入数据

查看日志：

```bash
docker compose logs -f
```

确认 compose 中有：

```yaml
environment:
  DATABASE_URL: file:/app/data/db.sqlite
volumes:
  - company-data:/app/data
```

## 安全建议

1. 首次部署后修改默认密码。
2. 定期备份 `company-data` volume。
3. 如果公网访问，建议配置 HTTPS 和访问限制。
