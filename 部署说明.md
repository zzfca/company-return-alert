# BC 公司申报管理系统 - NAS Docker 部署指南

## 📋 项目信息
- **技术栈**: Next.js 15 + SQLite + Drizzle ORM
- **数据库**: SQLite（文件数据库，无需额外数据库服务）
- **访问方式**: 打开地址后直接进入系统

## 🐳 NAS 部署步骤

### 1️⃣ 准备工作

#### 在你的 Windows 电脑上：

1. **打包项目文件**
   ```bash
   cd "c:\Users\xie\code project\2nd-company"
   
   # 创建一个临时文件夹排除不需要的文件
   # 手动复制整个 2nd-company 文件夹到一个新位置，删除以下内容：
   # - node_modules 文件夹
   # - .next 文件夹
   # - db.sqlite 文件（会在 NAS 上重新生成）
   
   # 然后压缩成 zip 文件，命名为 bc-company-manager.zip
   ```

2. **确认需要的文件**（打包时确保包含）：
   - ✅ `Dockerfile`
   - ✅ `docker-compose.yml`
   - ✅ `.dockerignore`
   - ✅ `package.json` 和 `package-lock.json`
   - ✅ `next.config.ts`
   - ✅ `drizzle.config.ts`
   - ✅ `src/` 目录（所有源代码）
   - ✅ `public/` 目录（如果有）

### 2️⃣ 在群晖/威联通 NAS 上部署

#### 方式 A：使用 Docker Compose（推荐）

1. **上传项目到 NAS**
   - 登录 NAS 管理界面（如 DSM）
   - 使用 File Station 上传 `bc-company-manager.zip`
   - 建议放在 `/docker/bc-company-manager/` 目录
   - 解压文件

2. **通过 SSH 连接 NAS**
   ```bash
   ssh your-nas-username@nas-ip-address
   ```

3. **进入项目目录并启动**
   ```bash
   cd /volume1/docker/bc-company-manager  # 群晖路径示例
   # 或者
   cd /share/Container/bc-company-manager  # 威联通路径示例
   
   # 构建并启动容器
   docker-compose up -d
   ```

4. **查看日志确认启动**
   ```bash
   docker-compose logs -f
   ```

5. **访问应用**
   - 浏览器打开：`http://你的NAS的IP:3000`
   - 默认账户：`xie / xie123` 或 `admin / admin123`

#### 方式 B：使用 NAS 的 Docker UI（Container Manager）

**群晖 NAS (DSM 7.x)：**

1. 打开 **Container Manager**
2. 点击 **项目** → **新增**
3. 设置项目名称：`bc-company-manager`
4. 选择路径：上传的项目文件夹
5. 选择来源：**建立 docker-compose.yml**
6. 粘贴 `docker-compose.yml` 内容或使用现有文件
7. 点击 **完成** 开始构建

**威联通 NAS (Container Station)：**

1. 打开 **Container Station**
2. 点击 **创建** → **创建应用程序**
3. 输入名称：`bc-company-manager`
4. 粘贴 `docker-compose.yml` 内容
5. 点击 **验证** → **创建**

### 3️⃣ 数据持久化配置

容器会自动挂载以下目录：

```yaml
volumes:
  - ./data:/app/data          # 数据目录
  - ./db.sqlite:/app/db.sqlite # SQLite 数据库文件
```

**重要提示**：
- 首次启动时，数据库会自动初始化
- 务必定期备份 `db.sqlite` 文件
- 建议在 NAS 上启用文件版本控制或快照功能

### 4️⃣ 端口配置

默认端口：**3000**

如需修改端口，编辑 `docker-compose.yml`：
```yaml
ports:
  - "8080:3000"  # 将左边的端口改为你想要的端口
```

### 5️⃣ 反向代理配置（可选）

如果想通过域名访问，可以在 NAS 上配置反向代理：

**群晖 DSM：**
1. 控制面板 → **登录门户** → **高级** → **反向代理服务器**
2. 新增：
   - 来源：`公司管理系统.你的域名.com`
   - 端口：`443` (HTTPS) 或 `80` (HTTP)
   - 目的地：`localhost:3000`

**威联通：**
1. myQNAPcloud → **反向代理**
2. 添加规则，配置类似

### 6️⃣ 常用 Docker 命令

```bash
# 查看容器状态
docker ps

# 查看日志
docker logs bc-company-manager

# 停止容器
docker-compose down

# 重启容器
docker-compose restart

# 重新构建并启动
docker-compose up -d --build

# 进入容器内部（调试用）
docker exec -it bc-company-manager sh
```

### 7️⃣ 备份与恢复

**备份数据库：**
```bash
# 在 NAS 上
cd /volume1/docker/bc-company-manager
cp db.sqlite db.sqlite.backup-$(date +%Y%m%d)
```

**恢复数据库：**
```bash
# 停止容器
docker-compose down

# 恢复数据库文件
cp db.sqlite.backup-20260616 db.sqlite

# 重启容器
docker-compose up -d
```

### 8️⃣ 更新应用

当有新版本时：
```bash
# 1. 备份数据库
cp db.sqlite db.sqlite.backup

# 2. 停止并删除旧容器
docker-compose down

# 3. 上传新的代码文件（覆盖旧文件，保留 db.sqlite）

# 4. 重新构建并启动
docker-compose up -d --build
```

## 🔧 故障排查

### 容器无法启动
```bash
# 查看详细日志
docker-compose logs app

# 检查端口占用
netstat -tuln | grep 3000
```

### 数据库错误
```bash
# 进入容器检查数据库
docker exec -it bc-company-manager sh
ls -la /app/db.sqlite
```

### 权限问题
```bash
# 在 NAS 上修改文件权限
chmod 666 db.sqlite
chown 1001:1001 db.sqlite
```

2. 确认端口未被占用
