# BC 公司申报管理系统 - NAS Docker 部署指南

## 项目信息

- **技术栈**: Next.js 15 + SQLite + Drizzle ORM
- **数据库**: SQLite（文件数据库，无需额外数据库服务）
- **镜像**: `ghcr.io/zzfca/company-return-alert:latest`
- **访问方式**: 打开地址后直接进入系统

## 部署方式

现在不需要把源码上传到 NAS，也不需要在 NAS 上安装 npm 依赖或构建 Next.js。GitHub Actions 会在代码推送到 `main` 后自动构建 Docker 镜像并发布到 GitHub Container Registry。

### 方式 A：使用 Docker Compose（推荐）

1. 通过 SSH 连接 NAS：

```bash
ssh your-nas-username@nas-ip-address
```

2. 创建部署目录：

```bash
mkdir -p /volume1/docker/bc-company-manager
cd /volume1/docker/bc-company-manager
```

3. 创建 `docker-compose.yml`：

```bash
nano docker-compose.yml
```

4. 粘贴以下内容：

```yaml
services:
  app:
    image: ghcr.io/zzfca/company-return-alert:latest
    container_name: bc-company-manager
    restart: unless-stopped
    ports:
      - "3588:3000"
    volumes:
      - company-data:/app/data
    environment:
      NODE_ENV: production
      PORT: 3000
      DATABASE_URL: file:/app/data/db.sqlite
    healthcheck:
      test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:3000"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s

volumes:
  company-data:
```

5. 拉取镜像并启动：

```bash
docker compose pull
docker compose up -d
```

6. 查看日志：

```bash
docker compose logs -f
```

7. 访问应用：

```text
http://你的NAS的IP:3588
```

打开后直接进入系统，无需登录。

### 方式 B：使用 NAS 的 Docker UI

**群晖 NAS (DSM 7.x)：**

1. 打开 **Container Manager**。
2. 点击 **项目** → **新增**。
3. 设置项目名称：`bc-company-manager`。
4. 选择一个部署目录，例如 `/volume1/docker/bc-company-manager`。
5. 选择来源：**建立 docker-compose.yml**。
6. 粘贴上面的 Compose 内容。
7. 点击 **完成**，系统会自动拉取镜像并启动。

**威联通 NAS (Container Station)：**

1. 打开 **Container Station**。
2. 点击 **创建** → **创建应用程序**。
3. 输入名称：`bc-company-manager`。
4. 粘贴上面的 Compose 内容。
5. 点击 **验证** → **创建**。

## 数据持久化

SQLite 数据库保存在 Docker volume `company-data` 中，对应容器内路径 `/app/data/db.sqlite`。

```yaml
volumes:
  - company-data:/app/data
```

重要提示：

- 首次启动时，数据库会自动初始化。
- 更新镜像不会删除数据。
- 只有运行 `docker compose down -v` 才会删除 volume 和数据库。
- 建议定期备份 Docker volume 或导出数据库文件。

## 端口配置

默认访问端口是 **3588**，容器内部端口是 **3000**。

如需修改端口，编辑 `docker-compose.yml`：

```yaml
ports:
  - "8080:3000"
```

然后访问：

```text
http://你的NAS的IP:8080
```

## 反向代理配置（可选）

如果想通过域名访问，可以在 NAS 上配置反向代理。

**群晖 DSM：**

1. 控制面板 → **登录门户** → **高级** → **反向代理服务器**。
2. 新增规则：
   - 来源：`公司管理系统.你的域名.com`
   - 端口：`443` (HTTPS) 或 `80` (HTTP)
   - 目的地：`localhost:3588`

**威联通：**

1. myQNAPcloud → **反向代理**。
2. 添加规则，目的地指向 `localhost:3588`。

## 常用 Docker 命令

```bash
# 查看容器状态
docker ps

# 查看日志
docker compose logs -f

# 停止容器
docker compose down

# 重启容器
docker compose restart

# 拉取最新镜像
docker compose pull

# 使用最新镜像启动
docker compose up -d

# 进入容器内部（调试用）
docker exec -it bc-company-manager sh
```

## 更新应用

当 GitHub 发布新镜像后，在 NAS 的部署目录运行：

```bash
docker compose pull
docker compose up -d
```

这会更新应用代码，但保留 `company-data` 里的数据库。

## 故障排查

### 容器无法启动

```bash
docker compose logs app
```

### 端口被占用

```bash
netstat -tuln | grep 3588
```

修改 `docker-compose.yml` 里的端口映射，例如：

```yaml
ports:
  - "8080:3000"
```

### 镜像无法拉取

确认 NAS 能访问 GitHub Container Registry：

```bash
docker pull ghcr.io/zzfca/company-return-alert:latest
```

如果提示没有权限，需要在 GitHub 仓库的 Package 设置里把镜像可见性设为 public。

## 安全建议

1. 定期备份数据库。
2. 使用反向代理配置 HTTPS。
3. 限制访问 IP（例如仅允许内网访问）。
4. 定期更新 Docker 镜像。
